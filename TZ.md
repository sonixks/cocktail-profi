# Техническое Задание: «Cocktail Profi»
## 1. Цель проекта:
Сервис помогает пользователям находить новые идеи через случайную подборку, навигацию по категориям, а также подсказывает, какие коктейли можно приготовить из того, что уже есть у вас дома, а также советует, что докупить для расширения карты напитков.

## 2. Роли пользователей:
- **Гость:** доступ к общему каталогу со случайной подборкой коктейлей, просмотр карточки коктейля с подробным описанием (фото, ингредиенты, инструкция), фильтрация коктейлей по категориям. 
- **Авторизованный пользователь (Бармен-любитель):** все возможности гостя, а также доступ к разделу «Мой бар» (поиск и сохранение списка ингредиентов, которые есть в наличии), получение подборки коктейлей на основе сохраненных ингредиентов, добавление понравившихся рецептов в личный список «Избранное».
- **Администратор:** управление базой ингредиентов и категорий, загрузка рецептов из внешних источников, модерация контента.

## 3. Основные модели данных/сущности:
 1. Category (Категория): Группировка коктейлей для навигации.

    *Поля:*
    - name - Название категории
    - name_ru - Название категории на русском
 2. Ingredient (Ингредиент): Справочник продуктов.

    *Поля:*
    - name - Название ингредиента
    - name_ru - Название ингредиента на русском
 3. Cocktail (Коктейль): Карточка готового напитка.

    *Поля:*
    - category - Внешний ключ связь с категорией
    - name - Название коктейля
    - name_ru - Название коктейля на русском
    - instruction - Инструкция по приготовлению коктейля
    - instruction_ru - Инструкция по приготовлению коктейля на русском
    - is_alcoholic - Булевое значение алкогольный или нет
    - is_published - Булевое значение будет ли отображаться коктейль на страницах
    - image_url - Ссылка на картинки
    - external_id - Идентификатор коктейля из api

## 4. Ключевой функционал:
- **Случайная подборка:** При заходе на главную страницу пользователь видит сетку из 24 случайных карточек коктейлей. При обновлении страницы подборка полностью меняется, предлагая новые варианты.
- **Навигация по категориям:** Пользователь переходит на вкладку «Категории», выбирает «Кофе/Чай» и получает список всех кофейных или чайных коктейлей. Аналогично и с другими категориями.
- **Управление «Моим баром»:** Авторизованный пользователь вводит в поиск «Текила», находит нужный ингредиент и нажимает кнопку «Добавить». Ингредиент сохраняется в его профиле. Также его можно удалить из профиля, нажав на соответствующую кнопку.
- **Умный подбор:** На основе списка ингредиентов в «Мой бар» при нажатии на кнопку «Подбор коктейлей» система генерирует два результата: «Можно приготовить»: Коктейли, для которых у пользователя есть все ингредиенты; «Не хватает 1-2 ингредиентов»: Коктейли, для которых есть почти все, но не хватает 1-2 ингредиентов.
- **Избранное:** Пользователь сохраняет понравившиеся коктейли в персональную коллекцию для быстрого доступа.

## 5. Внешние интеграции и аналитика:
**Интеграция:** Использование внешнего API TheCocktailDB для загрузки и обновления базы рецептов.
